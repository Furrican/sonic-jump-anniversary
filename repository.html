Online Stages Instructions

This is the page where you can follow instructions on how to use the GitHub Repository and how to publish your own game maps.

If you encounter problems in the GitHub Repository, some of your answers can be found in the Q&A Page.

If you have entered this menu from the Online Stages link, you can go directly to the GitHub Repository via the link down below.

Online Stages

GitHub Repository



DESCRIPTION:
When selecting the Online Stages, you will enter in the custom map GitHub repository. This is where you can find all published game maps.

Each game map is located in pages known as "issues". These issues have the Custom Map label next to their title.

SEARCH FUNCTIONS:
If you want to search for a specific issue, you can use the search bar on top of the issues list.

By default, the Custom Map label is used in the search bar, but you can remove it to look into all other issues.
Removing the Custom Map label can be useful in case you want to find game maps inside issues that don't use this label.

You can also search the issues by authors by using the Authors tab.


Downloading a Map

If you want to play a game map, you need to download a game map.

The steps to download and play a game map are as followed:

  • Step 1: Choose one of the available issues by pressing on the issue title.
  • Step 2: Find the map file located in the issue description and select it to download the file.
  • Step 3: Once the download is complete, press BACK on the Android navigation bar to exit the repository and enter the Stage Select menu of the game.


Publishing your Map

PUBLISHING YOUR MAP:
If you want to publish your map in the Online Stages page, you need create a new issue in the GitHub repository.

The steps to publish your map are as followed:

  • Step 1: Select New Issue to enter the create New Issue menu.
  • Step 2: Insert a title. You can also write a description to better explain your map.
  • Step 3: Attach your map file by pressing Add files and and search for a map.bin or decoded_map.txt file located in the SonicJump subfolder. Your attached map file will appear in the description.
  • Step 4: If you have access to the Labels option, you can select the Custom Map label so your map will appear in the repository.
  • Step 5: Once the previous steps are done, press Save.
